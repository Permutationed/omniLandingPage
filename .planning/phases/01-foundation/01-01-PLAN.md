  <name>Task 2: Configure design tokens and globals.css</name>
  <files>
    app/globals.css

  </files>
  <action>
Replace the contents of app/globals.css with the complete design system for a light-mode fintech aesthetic. This includes:

1. Tailwind imports (@import "tailwindcss")
2. Base CSS variables in :root for all design tokens
3. @theme inline block to bridge CSS variables to Tailwind utilities
4. Base layer styles for body and button cursor

Use OKLCH color format (Tailwind v4 default). The primary color should be a premium purple/blue at hue 260.

Key tokens to define:
- Colors: background, foreground, card, popover, primary, secondary, muted, accent, destructive, border, input, ring
- Each color needs a foreground variant for text on that background
- Radii: sm, md, lg, xl, 2xl, full (derived from --radius base value of 0.625rem)
- Shadows: sm, md, lg

Also add font bridging for Inter (to be configured in layout):
```css
@theme inline {
  --font-sans: var(--font-inter), ui-sans-serif, system-ui, sans-serif;
}
```

Create lib/utils.ts with the cn() utility function:

```typescript
import { clsx, type ClassValue } from 'clsx'
import { twMerge } from 'tailwind-merge'

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs))
}
```

Include the button cursor fix for Tailwind v4:
```css
@layer base {
  button:not(:disabled) {
    cursor: pointer;
  }
}
```
  </action>
  <verify>
After saving, run the dev server and open browser DevTools. Inspect :root and confirm CSS variables are defined. Test that var(--primary) returns an oklch color value.
  </verify>
  <done>
Design tokens defined in CSS variables. @theme inline bridges them to Tailwind utilities. cn() utility function available.
  </done>
</task>
