# Plan 07.2-04: Component Styling & Spacing Refinement

## Objective

Refine component styling and spacing to match premium fintech standards. This includes:
- Reducing border radius for cleaner look
- Adding depth to cards through layered shadows
- Increasing whitespace in key areas
- Improving button hover states

## Research Basis

**Current issues**:
- Base radius 10px may be too rounded for fintech
- Cards lack depth layering
- Some sections feel cramped
- Button hover is just opacity change

**Competitive standards**:
- Bilt: 8-17px radius, layered transparency
- Mercury: 6-20px radius, frosted glass, generous padding
- Stripe: Subtle shadows, clean edges, lots of whitespace

## Tasks

<task id="1">
Reduce base border radius

File: `app/globals.css`

Change:
```css
--radius: 0.625rem;  /* 10px */
```

To:
```css
--radius: 0.5rem;  /* 8px */
```

This affects all calculated radius values (--radius-sm through --radius-xl).
</task>

<task id="2">
Improve shadow depth system

File: `app/globals.css`

Update shadow definitions for more depth:

```css
/* Current */
--shadow-sm: 0 1px 2px 0 oklch(0.20 0.02 270 / 0.08);
--shadow-md: 0 4px 6px -1px oklch(0.20 0.02 270 / 0.12), 0 2px 4px -2px oklch(0.20 0.02 270 / 0.08);
--shadow-lg: 0 10px 15px -3px oklch(0.20 0.02 270 / 0.15), 0 4px 6px -4px oklch(0.20 0.02 270 / 0.10);

/* Proposed - more pronounced, fintech-appropriate */
--shadow-sm: 0 1px 2px 0 oklch(0.20 0.01 270 / 0.06), 0 1px 3px 0 oklch(0.20 0.01 270 / 0.04);
--shadow-md: 0 4px 6px -1px oklch(0.20 0.01 270 / 0.08), 0 2px 4px -2px oklch(0.20 0.01 270 / 0.04);
--shadow-lg: 0 10px 25px -5px oklch(0.20 0.01 270 / 0.12), 0 8px 10px -6px oklch(0.20 0.01 270 / 0.06);
--shadow-xl: 0 20px 40px -10px oklch(0.20 0.01 270 / 0.15), 0 10px 20px -8px oklch(0.20 0.01 270 / 0.08);
```

Add xl shadow to theme block:
```css
--shadow-xl: var(--shadow-xl);
```
</task>
