# Plan 07.2-02: Typography System - Add Display Font

## Objective

Add a distinctive display font for headlines to differentiate from the generic "Inter everywhere" look. This creates visual hierarchy and brand identity comparable to fintech leaders.

## Research Basis

| Site | Display Font | Body Font |
|------|-------------|-----------|
| Bilt | GT America | Inter |
| Mercury | Arcadia Display | Arcadia |
| Ramp | Custom | System |
| **Omni (current)** | Inter | Inter |
| **Omni (proposed)** | Geist | Inter |

**Recommended: Geist** by Vercel
- Free and open source
- Modern, clean, slightly geometric
- Pairs excellently with Inter
- Used by premium tech companies
- Available via `next/font/google` or local

## Tasks

<task id="1">
Install Geist font via next/font

File: `app/layout.tsx`

Add Geist import alongside Inter:

```typescript
import { Inter } from 'next/font/google'
import { GeistSans } from 'geist/font/sans'

// Or if using next/font/local:
import localFont from 'next/font/local'

const geist = localFont({
  src: [
    { path: '../public/fonts/GeistVF.woff2', weight: '100 900' }
  ],
  variable: '--font-geist',
  display: 'swap',
})
```

Alternative: Use `@fontsource/geist-sans` npm package

Install: `npm install geist` (includes font files)
</task>

<task id="2">
Add font CSS variable to globals.css

File: `app/globals.css`

In the `@theme inline` block, add:

```css
--font-display: var(--font-geist), ui-sans-serif, system-ui, sans-serif;
--font-sans: var(--font-inter), ui-sans-serif, system-ui, sans-serif;
```

Add utility class:

```css
.font-display {
  font-family: var(--font-display);
}
```
</task>

<task id="3">
Update body tag with both font variables

File: `app/layout.tsx`

Update className on body or html:

```tsx
<body className={`${inter.variable} ${geist.variable} font-sans ...`}>
```
</task>

<task id="4">
Apply display font to hero headline

File: `components/hero/hero-content.tsx`

Change:
```tsx
<motion.h1 className="text-4xl sm:text-5xl md:text-6xl lg:text-7xl font-bold tracking-tight text-foreground">
```

To:
```tsx
<motion.h1 className="font-display text-4xl sm:text-5xl md:text-6xl lg:text-7xl font-bold tracking-tight text-foreground">
```
</task>

<task id="5">
Apply display font to section headings

Files:
- `components/how-you-earn/tuition-rewards/tuition-rewards.tsx`
- `components/how-you-earn/unlock-rate/unlock-rate.tsx`
- `components/how-you-earn/discover-rewards/discover-rewards.tsx`
- `components/how-you-earn/rewards-loop/rewards-loop.tsx`

Find all `<h2>` or main section headings and add `font-display` class.

Pattern:
```tsx
<h2 className="font-display text-3xl md:text-4xl lg:text-5xl font-bold ...">
```
</task>

<task id="6">
Optionally apply to metric card values

File: `components/hero/metric-cards.tsx`

Consider adding `font-display` to the value text:
```tsx
<div className="font-display text-2xl sm:text-3xl font-bold text-primary">{value}</div>
```

This creates visual consistency between headlines and key numbers.
</task>

## Typography Scale Reference

After changes, the hierarchy should be:

| Element | Font | Size | Weight |
|---------|------|------|--------|
| Hero H1 | Geist | 4xl-7xl | Bold |
| Section H2 | Geist | 3xl-5xl | Bold |
| Card Titles | Inter | lg-xl | Semibold |
| Body | Inter | base-lg | Normal |
| Labels | Inter | sm-xs | Medium |
